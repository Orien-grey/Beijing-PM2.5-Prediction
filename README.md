# 北京 PM2.5 浓度预测与特征工程优化项目

这是一个基于经典北京 PM2.5 数据集（2010-2014）的机器学习预测项目。本项目展示了如何通过系统性的**特征工程（Feature Engineering）**和时间序列分析，将一个极低性能的基线模型，优化为高精度预测模型的过程。

## 📊 模型版本与迭代对比

本项目共包含两个版本的代码，分别记录了算法优化的核心思路：

| 版本文件 | 核心特征 (Features) | 采用算法 (Model) | $R^2$ 分数 (满分 1.0) | MAE (平均绝对误差) | 结论与表现 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **v1_baseline** | 仅即时气象特征 (`TEMP`, `PRES`, `Iws`) | 线性回归 / 随机森林 | **0.0794** | 较大 | **翻车。** 模型无法捕捉空气污染的剧烈爆发，预测值呈一条直线。 |
| **v2_optimized** | 基础特征 + 时间特征 (`month`, `hour`) + 独热编码 (`cbwd`) + **前一小时历史浓度 (`pm2.5_last_hour`)** | 优化后的随机森林 (200棵树) | **0.9277** | **13.39 μg/m³** | **完美拟合。** 模型不仅能精准跟上每一次波峰波谷，还能高效预测重度污染。 |

## 💡 核心优化思路 (Key Inside)

1. **时间连续性依赖 (Temporal Dependency):** 空气污染具有极强的物理连续性。通过 `shift(1)` 操作构建了“前一小时浓度”特征，为模型提供了最关键的非线性推导线索。
2. **类别特征编码 (Categorical Encoding):** 原始数据中的风向 (`cbwd`) 为文本数据。项目采用**独热编码（One-Hot Encoding）**将其转换为高维稀疏数字矩阵，使其兼容随机森林的矩阵运算。
3. **超参数调优 (Hyperparameter Tuning):** 在优化版本中，将随机森林的基分类器数量（`n_estimators`）提升至 200 棵，并限制最大深度（`max_depth=20`）以兼顾泛化能力、防止过拟合。

## 📁 文件结构说明
* `beijing_pm2.5_v1_baseline.py`: 初始低分基线模型代码。
* `beijing_pm2.5_v2_optimized.py`: 引入特征工程后的高分优化模型代码。
